# 普惠贷款Excel表如何结合AI使用

## 一、场景背景

系统导出的Excel表包含以下字段：

| 字段 | 含义 |
|---|---|
| 企业名称 | 客户主体 |
| 普惠贷款今日余额 | 当前贷款余额 |
| 普惠贷款昨日余额 | 昨日贷款余额 |
| 普惠贷款上月余额 | 上月对比余额 |
| 普惠贷款年初余额 | 年初余额 |
| 授信 | 客户授信额度 |

这类数据在银行普惠金融业务中，可以用于**余额监测、客户经营、授信使用分析、风险预警和客户经理跟进管理**。

---

## 二、可以衍生的关键指标

在原始Excel基础上，建议增加以下计算字段：

| 指标 | 计算方式 | 业务含义 |
|---|---|---|
| 较昨日增减 | 普惠贷款今日余额 - 普惠贷款昨日余额 | 判断日度余额变动 |
| 较上月增减 | 普惠贷款今日余额 - 普惠贷款上月余额 | 判断月度增长或压降 |
| 较年初增减 | 普惠贷款今日余额 - 普惠贷款年初余额 | 判断全年贡献 |
| 授信使用率 | 普惠贷款今日余额 / 授信 | 判断客户额度使用程度 |
| 剩余可用额度 | 授信 - 普惠贷款今日余额 | 判断后续投放空间 |

### Excel公式示例

假设字段位置如下：

| 列 | 字段 |
|---|---|
| A列 | 企业名称 |
| B列 | 普惠贷款今日余额 |
| C列 | 普惠贷款昨日余额 |
| D列 | 普惠贷款上月余额 |
| E列 | 普惠贷款年初余额 |
| F列 | 授信 |

可新增字段：

```excel
较昨日增减 = B2-C2
较上月增减 = B2-D2
较年初增减 = B2-E2
授信使用率 = B2/F2
剩余可用额度 = F2-B2
```

## 三、AI可以帮助银行做什么

### 1. 自动生成普惠贷款日报

每天上传Excel后，AI可以自动生成日报，包括：

- 普惠贷款总余额；
- 较昨日增减；
- 较上月增减；
- 较年初增减；
- 增长客户TOP10；
- 下降客户TOP10；
- 授信使用率较高客户；
- 授信使用率较低客户；
- 零余额但有授信客户；
- 客户经理后续跟进建议。

### 示例输出

> 今日普惠贷款余额较昨日增加XX万元，较上月增加XX万元，较年初增加XX万元。增长主要来自A公司、B公司、C公司；下降较明显客户为D公司、E公司。建议重点关注授信使用率低但授信额度较高的客户，督促客户经理跟进提款需求；同时对余额下降明显客户排查是否存在还款、客户流失或风险压降情况。

---

### 2. 自动识别重点客户

AI可以根据余额变化和授信使用率，将客户分为不同类型：

| 客户类型 | 判断逻辑 | 管理建议 |
|---|---|---|
| 高使用率客户 | 授信使用率 > 80% | 关注续贷、增信、额度调整 |
| 低使用率客户 | 授信使用率 < 30% | 客户经理跟进提款需求 |
| 余额快速增长客户 | 较上月明显增加 | 识别新增投放贡献 |
| 余额明显下降客户 | 较上月明显减少 | 排查还款、流失、压降原因 |
| 零余额有授信客户 | 今日余额 = 0 且授信 > 0 | 重点营销提款 |
| 超授信或异常客户 | 今日余额 > 授信 | 核查数据或风险情况 |

---

### 3. 自动生成客户经理跟进清单

AI可以输出类似以下清单：

| 企业名称 | 当前余额 | 授信 | 授信使用率 | 建议动作 |
|---|---:|---:|---:|---|
| A公司 | 800万 | 1000万 | 80% | 关注续贷和额度是否充足 |
| B公司 | 0 | 500万 | 0% | 客户经理跟进提款需求 |
| C公司 | 300万 | 1000万 | 30% | 了解企业后续资金使用计划 |
| D公司 | 较上月下降200万 | 800万 | - | 排查是否存在客户流失或风险压降 |

## 四、建议补充的字段

目前字段适合做余额变动和授信使用分析。若要进一步提升AI分析质量，建议系统导出时增加以下字段：

| 建议字段 | 用途 |
|---|---|
| 客户经理 | 按客户经理统计业绩和跟进责任 |
| 所属支行/网点 | 按机构统计排名和贡献 |
| 行业分类 | 分析行业集中度和行业风险 |
| 贷款到期日 | 做续贷和到期预警 |
| 贷款利率 | 分析收益贡献 |
| 贷款品种 | 区分信用贷、抵押贷、担保贷等 |
| 企业规模 | 区分小微、个体工商户等 |
| 是否首贷户 | 支持普惠考核分析 |
| 是否续贷客户 | 判断客户稳定性 |
| 风险分类/五级分类 | 风险监测必备 |
| 逾期天数 | 识别风险客户 |
| 担保方式 | 判断风险缓释能力 |

## 五、可直接使用的AI提问模板

### 1. 生成普惠贷款日报

```text
请根据这张普惠贷款余额表，生成一份银行内部日报，包括总余额、较昨日变化、较上月变化、较年初变化、增长前10客户、下降前10客户、授信使用率分析和管理建议。
```

### 2. 筛选营销客户

```text
请帮我筛选授信使用率低于30%、但授信额度大于500万元的企业，并生成客户经理营销跟进清单。
```

### 3. 做风险预警

```text
请帮我找出余额较上月下降超过30%的客户，并分析需要关注的风险点。
```

### 4. 做客户分类

```text
请按照授信使用率、余额增长、余额下降，把客户分为重点维护客户、潜力提款客户、风险关注客户和一般客户。
```

### 5. 生成行长汇报材料

```text
请把这张表整理成一段适合支行行长汇报的文字，语言正式、简洁，突出普惠贷款投放情况、客户变化情况和下一步措施。
```

## 六、推荐落地方式

### 方案一：每日上传Excel，让AI生成日报

适合刚开始使用AI的阶段。

流程：

1. 从系统导出Excel；
2. 上传给AI；
3. AI计算关键指标；
4. AI识别重点客户；
5. AI生成日报和客户经理跟进清单。

---

### 方案二：制作固定Excel模板

可以建立一个固定分析模板，包括：

- 指标计算公式；
- 条件格式；
- 客户分类规则；
- 透视表；
- 图表；
- 自动日报区域；
- 客户经理跟进清单。

以后每天只需要粘贴系统导出的数据，即可自动生成分析结果。

---

### 方案三：建设普惠贷款经营看板

如果行内系统和合规条件允许，可以建设经营看板，展示：

- 今日普惠贷款余额；
- 较昨日、较上月、较年初变化；
- 授信使用率分布；
- 增长客户TOP10；
- 下降客户TOP10；
- 零余额有授信客户；
- 高授信低使用客户；
- 客户经理排名；
- 支行排名；
- 到期预警客户；
- 行业集中度分析。

## 七、银行使用AI的合规注意事项

银行数据敏感，使用AI时建议：

1. 不上传客户证件号、统一社会信用代码、手机号、账户号等敏感信息。
2. 企业名称如涉及敏感客户，可脱敏为“客户A、客户B”。
3. 不上传内部评级、征信细节、审批意见等高度敏感字段，除非使用的是行内合规批准的AI环境。
4. AI输出的风险判断只能作为辅助，不能直接替代信贷审批、风险分类或授信决策。
5. 对外报送和考核口径必须以行内制度和核心系统数据为准。
6. 涉及客户隐私、信贷审批、监管报送的数据处理，应遵守本行数据安全和保密制度。

## 九、结论

这张普惠贷款Excel表结合AI后，可以从普通报表升级为银行普惠业务的经营分析工具。

核心价值包括：

- 自动计算余额变动；
- 自动识别重点客户；
- 自动筛选营销对象；
- 自动发现异常和风险线索；
- 自动生成日报、汇报材料和客户经理跟进清单；
- 支持支行、部门和客户经理进行精细化管理。

初期建议先做“Excel + AI日报”模式，稳定后再升级为固定模板或经营看板。
